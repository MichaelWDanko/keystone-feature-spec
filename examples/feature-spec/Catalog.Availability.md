# Catalog.Availability

Defines requirements for showing whether a tool can be borrowed.

## Requirements

- Availability MUST distinguish available, on-loan, reserved, and
  reference-only items.
- An on-loan item MUST show its expected return date when one is known.
- Availability changes MUST appear without requiring users to start a new
  catalog search.
- An availability failure MUST communicate that the displayed state may be
  outdated.
