Keep the vision when the code changes.

Keystone is a Markdown-native framework for documenting feature requirements outside the implementation. Requirements stay visible when code is refactored, replaced, or removed.

See how it works Read the framework specification →

A green test suite can still miss a feature.

Tests show that the implementation behaves as the tests expect. They cannot show whether the suite still covers every behavior a feature requires.

An agent can remove a feature, remove its related tests, and leave the suite passing. Keystone keeps the required behavior in a separate, reviewable file, so that change becomes explicit instead of disappearing as cleanup.

Tests answer Does the code behave as expected?
Keystone answers Does this still do what the feature requires?

Feature requirements your whole project can read.

Keystone keeps requirements close to the codebase without binding them to a vendor or a second data format.

Readable by people and agents

Requirements are ordinary Markdown. They work in code review, version control, a text editor, and the context window of an AI coding agent.

A durable requirement Removing one account MUST NOT modify another account.

Independent of implementation

The requirements describe what must remain true, not classes, files, or a particular approach. A rewrite does not erase the feature definition.

Not an implementation detail Users MUST be able to recover a configured account.

Designed for careful change

Changes become easier to review because the intended behavior is explicit. When a feature changes, update its requirements on purpose.

An intentional feature change Update the requirement Reason: Replaced by a new feature.

Structure makes broad promises hard to lose.

Related behavior stays connected through a simple namespace. The detailed guide explains how files inherit requirements and how to document a real exception.

Small framework. Clear requirements.

Keystone is an early framework proposal. Its namespace, inheritance, status, and exception rules are usable now. The dependency-free validator checks document structure and references.

Coverage discovery and effective-spec rendering are on the roadmap.
