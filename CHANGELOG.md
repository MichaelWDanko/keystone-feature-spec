# Changelog

All notable changes to Keystone are documented in this file.

## [0.2.0] - 2026-07-24

### Added

- Add `TODO.*.md` documents for defining product behavior that has not yet been
  implemented.
- Allow active and TODO documents to describe the same target namespace while
  keeping current and intended behavior distinct.
- Validate TODO filenames, target headings, related references, and exception
  sources.

### Changed

- Reframe Keystone as a way to define the intended shape of a whole product or
  selected features, with preservation as a benefit of active specifications.
- Clarify that Keystone defines product intent and implementation state without
  managing a backlog or deciding what an agent may implement.

## [0.1.0] - 2026-07-20

### Added

- Publish the initial draft contract, validator, agent setup prompt, examples,
  and documentation site.
